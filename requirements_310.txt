# 额外环境配置:
# 1. 需要准备 pubg_test 工程，并确保存在:
#    pubg_test/control_proxy/src/gametest_proxy/
# 2. 默认自动查找位置是 auto_game 同级目录下的 pubg_test；也可以设置:
#    export PUBG_TEST_PATH=/path/to/pubg_test
# 3. 开启搜房:
#    export AUTOGAME_PUBG_ROOM_SEARCH=1
#    或在 aw/autogame/config/config.json 的 pubg_room_search.enabled 配置为 true
# 4. mixed_yolo 模式需要可访问 YOLO 服务，默认:
#    yolo_host=localhost, yolo_port=6666
# 5. replay/dsl_replay 或门校准使用 sam3 时，需要可访问 SAM3 服务，默认:
#    sam3_host=localhost, sam3_port=12345
# 6. gametest_proxy 来自 pubg_test/control_proxy/src，不是 PyPI 包，不能通过 pip install 安装。
#
# 远程大脑服务器 remote_brain
# 代码位置: remote_brain/
# 启动命令:
#    cd /path/to/auto_game
#    python3 -m remote_brain.server --host 0.0.0.0 --port 8765
# 健康检查:
#    curl http://服务器IP:8765/health
#
# 服务器 pip 依赖:
# 1. remote_brain/server.py 使用 Python 标准库 http.server，不需要 fastapi/flask/uvicorn/requests。
# 2. 图像编解码和算法推理依赖已由上面的 numpy/opencv-python/torch/torchvision/Pillow/scikit-learn/ultralytics 覆盖。
# 3. 如果服务器没有桌面环境，可以把 opencv-python 改成 opencv-python-headless，二者二选一，不要同时安装。
#
# 服务器运行前必须准备:
# 1. 上传完整 auto_game 仓库，而不是只上传 remote_brain 目录。
# 2. 确认模型权重文件存在:
#    aw/autogame/customs_examples/Auto_PUBG_ALL/resource/weights/
# 3. 默认项目/用例:
#    TARGET_PROJECT_CASE=Auto_PUBG_ALL
#    TARGET_GAME_CASE=auto_pubg
#    remote_brain 会按请求自动设置；手动调试时也可自行 export。
# 4. 分辨率由本地 client 每帧通过 screen.width/screen.height 传给服务器；
#    无 hdc 的服务器不需要连接设备。
# 5. 如果启用搜房，服务器同样需要上方 pubg_test、YOLO 服务、SAM3 服务配置。
#
